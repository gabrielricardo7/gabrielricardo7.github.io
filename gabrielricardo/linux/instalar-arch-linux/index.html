Instalar Arch Linux

Linux / Por Gabriel Ricardo
Linux

Arch Linux

O Arch Linux é uma distribuição GNU/Linux x86_64 de uso geral desenvolvida independentemente que se empenha em fornecer as últimas versões estáveis da maioria dos softwares seguindo um modelo de lançamento contínuo (rolling-release). A instalação padrão é um sistema base mínimo, configurado pelo usuário para adicionar apenas o que é propositalmente necessário. É por isso que instalar o Arch Linux é um desafio em si, mas ao mesmo tempo, é uma oportunidade de aprendizado para usuários intermediários do Linux.

Eu sou um completo iniciante no GNU/Linux. Eu deveria usar o Arch?
Se você é um iniciante e deseja usar o Arch, você deve estar disposto a investir tempo para aprender um novo sistema, e aceitar que o Arch é projetado como uma distribuição de faça você mesmo; é o usuário que monta o sistema.

ArchWiki

Vou mostrar como instalar o Arch Linux. Por favor, siga os passos cuidadosamente e leia as instruções corretamente.

As etapas de instalação podem diferir em alguns pontos, dependendo se você possui um sistema UEFI ou BIOS herdado. A maioria dos novos sistemas vem com UEFI atualmente.

Aviso!
O método discutido aqui apaga os sistemas operacionais existentes no seu computador e instala o Arch Linux nele. Portanto, se você seguir este tutorial, certifique-se de ter feito backup dos arquivos ou perderá tudo. Você foi avisado.

Porém, antes de ver como instalar o Arch Linux a partir de um USB, verifique se você possui os seguintes requisitos:

Requisitos para instalar o Arch Linux:

  • Uma máquina compatível com x86_64 (ou seja, 64 bits)
  • Mínimo de 512 MB de RAM (recomendado 2 GB)
  • Pelo menos 2 GB de espaço livre em disco (recomendado 20 GB para uso básico em um ambiente de desktop)
  • Uma conexão ativa à Internet
  • Uma unidade USB com capacidade mínima de 2 GB de armazenamento
  • Familiaridade com a linha de comando do Linux

Passo 0: Baixar o Arch Linux (ISO)

Você pode baixar a ISO a partir do site oficial. Estão disponíveis links diretos para download e torrent.

Download

Passo 1: Criar um USB-executável do Arch Linux

Você precisará criar um USB-executável do Arch Linux a partir da ISO que você acabou de baixar.
Você pode usar a ferramenta GUI do Etcher para criar o USB-executável. Está disponível para Windows e Linux.

balenaEtcher

Passo 2: Iniciar a partir do USB-executável

Observe que, em alguns casos, talvez não seja possível inicializar a partir do USB-executável com a inicialização segura ativada. Se for esse o seu caso, desative a inicialização segura primeiro.
Depois de criar um USB-executável para o Arch Linux, desligue o seu PC. Conecte seu USB e inicialize seu sistema. Durante a inicialização, pressione as teclas F2, F10 ou F12 (dependendo do sistema) para entrar nas configurações de inicialização.
Aqui, selecione para inicializar a partir de USB ou disco removível. Depois de fazer isso e o sistema inicializar, você verá uma opção como esta:

Selecione Boot Arch Linux (x86_64). Após várias verificações, o Arch Linux inicializa no prompt de login com o usuário root.

Não está usando teclado americano? Leia isso:
O layout do teclado padrão na sessão live é US. Embora a maioria dos teclados em inglês funcione perfeitamente, o mesmo não ocorre com os teclados francês, alemão e outros.
Se você enfrentar dificuldades, poderá listar todo o layout do teclado suportado:

ls /usr/share/kbd/keymaps/*/.map.gz

E então altere o layout para o apropriado usando o comando loadkeys. Por exemplo, se você deseja teclado brasileiro, é isso que você usará:

loadkeys br-abnt2

Passo 3: Conectar ao WiFi

Toda a instalação do Arch precisa de acesso à internet, por isso é importante que você esteja conectado.

Caso você precise configurar a sua rede cabeada, confira estas informações.

Você pode se conectar ao Wi-Fi interativamente usando este utilitário útil chamado wifi-menu. Basta digitar este comando e siga as etapas:

wifi-menu

Você deve poder ver as conexões ativas e se conectar a elas usando a senha. Uma vez conectado, verifique se você pode usar a Internet usando o comando ping:

ping google.com

Se você receber bytes em resposta, estará conectado. Use Ctrl+C para interromper a resposta do ping.

Os próximos passos incluem particionar disco, criar o sistema de arquivos e montá-lo.
Novamente, leia todas as instruções corretamente e siga cada uma das etapas cuidadosamente. Você perde um passo ou ignora algo e terá dificuldade em instalar o Arch.

Passo 4: Particionar os discos

Para particionar os discos, usaremos o gerenciador de partições baseado em linha de comando fdisk.
Use este comando para listar todo o disco e partições no seu sistema:

fdisk -l

Seu disco rígido deve estar rotulado como /dev/sda ou /dev/nvme0n1. Por favor, use a etiqueta de disco apropriada para o seu sistema. Estou usando /dev/sda porque isso é mais comum.
Primeiro, selecione o disco que você irá formatar e particionar:

fdisk /dev/sda

Sugiro que você exclua todas as partições existentes no disco usando o comando d. Depois de ter todo o espaço livre em disco, é hora de criar novas partições com o comando n.

Verifique se você tem o modo UEFI* ativado

Algumas etapas são diferentes para sistemas UEFI e não UEFI. Você deve verificar se possui ou não o sistema habilitado para UEFI. Use este comando:

ls /sys/firmware/efi/efivars

Se esse diretório existir, você possui um sistema habilitado para UEFI. Você deve seguir as etapas para o sistema UEFI. Os passos que diferem são mencionados claramente.

*Crie uma partição ESP (apenas para sistemas UEFI)

Se você possui um sistema UEFI, deve criar uma partição EFI no início do seu disco. Caso contrário, pule esta etapa.
Quando você digitar n, ele solicitará que você escolha um número de disco, digite 1. Permaneça com o tamanho padrão do bloco; quando solicitar o tamanho da partição, digite +512M.

Uma das etapas importantes é alterar o tipo da partição EFI para EFI System (em vez do sistema Linux).
Digite t para alterar o tipo. Digite L para ver todos os tipos de partição disponíveis e, em seguida, insira o número correspondente no sistema EFI.

Criar partição raiz (root)

Você precisa criar uma partição raiz para os sistemas UEFI e legado.
A prática comum de particionamento era/é criar partições raiz, swap e home separadamente. Você pode apenas criar uma única partição raiz e criar um arquivo de troca e home no próprio diretório raiz.
Portanto, nessa abordagem, teremos uma única partição raiz, sem swap, sem home.
Enquanto você estiver no comando fdisk, pressione n para criar uma nova partição. Ele fornecerá automaticamente o número de partição 2. Desta vez, mantenha pressionado Enter para alocar todo o espaço restante em disco para a partição raiz.

Quando você terminar o particionamento de disco, insira o comando w para gravar as alterações no disco e sair do comando fdisk.

Passo 5: Criar sistema de arquivos

Agora que você já tem suas partições de disco prontas, é hora de criar um sistema de arquivos. Siga as etapas para o seu sistema

Criando sistema de arquivos para o sistema UEFI

Portanto, você tem duas partições de disco e a primeira é do tipo EFI. Crie um sistema de arquivos FAT32 nele:

mkfs.fat -F32 /dev/sda1

Agora crie um sistema de arquivos Ext4 na partição raiz:

mkfs.ext4 /dev/sda2

Criando sistema de arquivos para sistema não UEFI

Para sistemas não UEFI, você possui apenas uma única partição raiz. Então, basta torná-lo ext4:

mkfs.ext4 /dev/sda1

Passo 6: Selecione um espelho apropriado

Este é um grande problema com a instalação do Arch Linux. Se você continuar instalando, poderá achar que os downloads são muito lentos. Em alguns casos, é tão lento que o download falha.
Isso ocorre porque a lista de espelhos (localizada em /etc/pacman.d/mirrorlist) possui um grande número de espelhos, mas não em boa ordem. O espelho superior é escolhido automaticamente e nem sempre é uma boa escolha.
Felizmente, há uma correção para isso. Primeiro, sincronize o repositório pacman para poder baixar e instalar o software:

pacman -Syy

Agora, instale também o reflector que você pode usar para listar os espelhos novos e rápidos localizados no seu país:

pacman -S reflector

Faça um backup da lista de espelhos (apenas no caso):

cp /etc/pacman.d/mirrorlist /etc/pacman.d/mirrorlist.bak

Agora, obtenha a boa lista de espelhos com reflector e salve-a na lista de espelhos. Você pode alterar o país de BR para seu próprio país.

reflector -c "BR" -f 12 -l 10 -n 12 --save /etc/pacman.d/mirrorlist

Tudo certo para continuar.

Passo 7: Instalar o Arch Linux

Como você tem tudo pronto, é hora de finalmente instalar o Arch Linux. Você o instalará no diretório raiz, então monte-o primeiro.
Você se lembra do nome da partição raiz? Use-o para montá-lo:

mount /dev/sda2 /mnt

Com o root montado, é hora de usar o maravilhoso script pacstrap para instalar todos os pacotes necessários:

pacstrap /mnt base linux linux-firmware vim nano

Levará algum tempo para baixar e instalar esses pacotes. Se os downloads forem interrompidos, não será necessário entrar em pânico. Você pode executar o comando acima mais uma vez e reiniciar o download.
Adicionei o editor de texto Vim e Nano à lista porque você precisará editar alguns arquivos após a instalação.

Passo 8: Configurar o sistema Arch instalado

Gere um arquivo fstab para definir como as partições de disco, dispositivos de bloco ou sistemas de arquivos remotos são montados no sistema de arquivos.

genfstab -U /mnt >> /mnt/etc/fstab

Agora use o arch-chroot e entre no disco montado como root. Na verdade, agora você está usando o sistema Arch Linux recém-instalado no disco. Você precisará fazer algumas alterações na configuração do sistema instalado para poder executá-lo corretamente ao inicializar a partir do disco.

arch-chroot /mnt

Definir fuso horário

Para configurar o fuso horário no Linux, você pode usar o comando timedatectl. Primeiro encontre seu fuso horário:

timedatectl list-timezones

E configure-o assim (substitua America/Sao_Paulo pelo fuso horário desejado):

timedatectl set-timezone America/Sao_Paulo

Podemos também sincronizar o relógio com as informações da BIOS, se ela estiver correta, o seu relógio também estará, ainda que todo mundo use o horário da internet praticamente hoje em dia:

 hwclock --systohc

Configurando localidade

É isso que define os formatos de idioma, numeração, data e moeda do seu sistema.
O arquivo /etc/locale.gen contém todas as configurações locais e o idioma do sistema em um formato comentado.
Abra o arquivo usando o editor Vim ou Nano e remova o comentário (remova o # do início da linha) do idioma que você preferir. Eu usei pt_BR.UTF-8 (português Brasil).
Agora gere a configuração da localidade no arquivo de diretório /etc usando os comandos abaixo, um por um:

locale-gen

echo LANG=pt_BR.UTF-8 > /etc/locale.conf

export LANG=pt_BR.UTF-8

As configurações de localidade e fuso horário podem ser alteradas mais tarde, quando você estiver usando seu sistema Arch Linux.

Configuração de rede

Crie um arquivo /etc/hostname e inclua a entrada de nome do host nesse arquivo. Hostname é basicamente o nome do seu computador na rede.
No meu caso, definirei o nome do host como myarch. Você pode escolher o que quiser:

echo myarch > /etc/hostname

A próxima parte é criar o arquivo hosts:

touch /etc/hosts

E edite esse arquivo /etc/hosts com o editor Vim ou Nano para adicionar as seguintes linhas (substitua myarch pelo nome do host escolhido anteriormente):

127.0.0.1 localhost.localdomain localhost
::1 localhost.localdomain localhost
127.0.1.1 myarch.localdomain myarch

Configurar senha de root

Você também deve definir a senha da conta root usando o comando passwd:

passwd

Digite duas vezes a sua nova senha e não a esqueça! (Anote-a se for necessário)

Você pode criar outros usuários se você quiser agora, mas isso também pode ser feito depois, pela interface, o que acaba sendo mais simples. De toda forma, como exemplo, você pode criar um ususário com o seu nome:

useradd -m -g users -G wheel nome_desejado_para_o_usuario

Vamos aproveitar a ocasião para instalar alguns pacotes que serão úteis na pós instalação do sistema:

pacman -S dosfstools os-prober mtools network-manager-applet networkmanager wpa_supplicant wireless_tools dialog sudo

Caso você já tenha criado outro usuário, e tenha instalado o “sudo” como eu fiz, é preciso adicionar o seu usuário dentro do arquivo “sudoers”:

 nano /etc/sudoers

Ao final do arquivo adicione “USER_NAME   ALL=(ALL) ALL”, por exemplo:

gabriel ALL=(ALL) ALL

Salve e saia.

Você pode criar um usuário usando o “useradd” normal, sem usar parâmetros, como você faria em outras distros, porém, esse comando “cru” no Arch Linux, não cria a sua pasta de usuário na home, então você pode criar ela manualmente, por exemplo:

mkdir /home/gabriel

E então mude a home para lá com:

usermod -d /home -m gabriel

Passo 9: Instalar o gerenciador de inicialização Grub

Instale o grub em sistemas UEFI

Essa é uma das etapas cruciais e difere para sistemas UEFI e não UEFI. Deixe-me mostrá-lo primeiro para os sistemas UEFI.
Verifique se você ainda está usando o arch-chroot. Instale os pacotes necessários:

pacman -S grub efibootmgr

Crie o diretório em que a partição EFI será montada:

mkdir /boot/efi

Agora, monte a partição ESP que você criou

mount /dev/sda1 /boot/efi

Instale o grub assim:

grub-install --target=x86_64-efi --bootloader-id=GRUB --efi-directory=/boot/efi

Um último passo:

grub-mkconfig -o /boot/grub/grub.cfg

Instale o grub em sistemas não UEFI

Instale o pacote grub primeiro:

pacman -S grub

E instale o grub assim (não coloque o número do disco sda1, apenas o nome do disco sda):

grub-install /dev/sda

Último passo:

grub-mkconfig -o /boot/grub/grub.cfg

Chegamos ao final da instalação padrão, digite “exit” ou pressione “Ctrl+D” e use o comando “reboot” para reiniciar o computador, remova o USB-executável da máquina.

Passo 10: Instalar um ambiente de área de trabalho

Depois da instalação

Depois de reiniciar o seu computador, já com o Arch Linux instalado no seu HD/SSD, logue-se com o usuário root para fazer mais algumas modificações, o nome do usuário é “root”, sem aspas, e a senha, é a que você definiu com o passwd.

Precisamos conectar o nosso sistema à internet novamente, mais uma vez, você pode usar o wifi-menu para conectar a uma rede wireless, ou então, usar o networkmanager que instalamos antes:

wifi-menu

systemctl status NetworkManager

systemctl start NetworkManager

Com a sua internet funcionando, vamos instalar alguns pacotes básicos para o funcionamento de qualquer interface, vamos atualizar o sistema e instalar o Xorg inicialmente:

pacman -Sy

pacman -S xorg-server

Vamos instalar agora os drivers de vídeo:

#Intel

pacman -S xf86-video-intel libgl mesa

#Nvidia

pacman -S nvidia nvidia-libgl mesa

#AMD

pacman -S mesa xf86-video-amdgpu

Caso você esteja usando o Arch em uma máquina virtual, como o Virtualbox, instale estes pacotes:

pacman -S virtualbox-guest-utils virtualbox-guest-modules-arch mesa mesa-libgl

Daqui por diante, depende um pouco do que você vai querer instalar como interface, se vai ser GNOME, KDE Plasma, XFCE, etc.

#GNOME

pacman -S gdm

systemctl enable gdm

Observe que neste caso, grupos de pacotes (meta-pacotes) como o “gnome”, instalam um conjunto amplo de aplicações (semelhante ao que o Fedora Linux entrega por padrão), se você quiser uma instalação mais enxuta, selecione os pacotes manualmente, como “gnome-shell”, “gedit”, etc. 

pacman -S gnome gnome-terminal nautilus gnome-tweaks gnome-control-center gnome-backgrounds adwaita-icon-theme

Você também pode querer instalar um outro navegador ou qualquer outro pacote, como o Firefox:

pacman -S firefox

#KDE Plasma

pacman -S sddm

systemctl enable sddm

pacman -S plasma konsole dolphin

#XFCE (instalado com GDM ou SDDM)

 pacman -S xfce4 xfce4-goodies xfce4-terminal

Para outros ambientes, mais uma vez, consulte a ArchWiki. Independente de qual ambiente você escolha, ative agora o Network Manager no boot do sistema para que a sua internet volte funcionando 100%.

systemctl enable NetworkManager

Agora é só reiniciar o sistema.

Se tudo deu certo, você tem um novo Arch Linux, logue-se via interface com o seu usuário root e aproveite para criar um novo usuário tradicional para que você use o sistema confortavelmente sem problemas de segurança, lembre-se de adicioná-lo ao arquivo sudoers, se quiser usar o sudo para rodar os comandos.

Você já deve ter percebido que instalar o Arch Linux não é tão fácil quanto instalar o Ubuntu. No entanto, com um pouco de paciência, você certamente pode realizá-lo e depois dizer ao mundo que usa o Arch Linux.

A instalação do Arch Linux propicia bastante aprendizado. Você pode continuar “brincando” e ver o quão poderoso é o Arch.
